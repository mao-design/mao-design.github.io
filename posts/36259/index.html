搭建准备

如果博客创建失败，那么就把自己的博客文件夹删掉重新配置就可以了，本Hexo博客是基于Gitee搭建的

首先要安装Git、Note.js、Python

下载地址：git

Git网址：https://git-scm.com/ 默认会自动配置环境变量

​ 检查是否安装成功：git -v

Node.js网址:http://nodejs.cn/ 默认会自动配置环境变量

​ 检查是否安装成功：在终端输入node -v

Python:https://www.python.org/ 安装Hexo不需要Python，但是有些主题的有些插件需要Python，所以安不安装都可以

检查是否安装成功：在终端输入Python，要退出输入 exit() 即可退出

安装Git上传插件

在自己本地博客的根目录下打开cmd终端输入 cnpm install hexo-deployer-git --save 下载插件，如果不下载这个插件的话，就不能使用 hexo d 命令推送自己的博客到服务器上

部署到仓库之前的准备

因为Gitee Pages维护升级的缘故，所以Gitee Pages暂停对私有仓库的开通支持。当前已开通 Pages 服务的仓库更新不受影响，所以如果要使用Gitee Pages服务的话，需要把自己的仓库改为公有仓库！！！

注意：如果仓库不设置为开源，则无法使用Gitee Pages服务！！！

一般每个主题都会有一个GitHub仓库以及配置文档，上面有详细的安装方法，以及要安装的插件，每个主题安装的方法略有不同，但是大差不差，基本流程：

  1. 在自己博客的根目录下，把下载好的主题包 解压 并移到一个名为 themes 的文件夹下，然后在回到根目录下，编辑一个叫 _config.yml 的配置文件，打开这个文件，找到 theme 选项，输入你要配置的主题的名字，注意 选项与名字之间要有空格，例：theme: 主题名称

博客上无法显示图片

首先在 blog/source/_posts目录下创建一个文件夹，专门用来放图片的，然后在创建一个md文件，而且 md文件的名字 要和 图片文件夹的名字 相同

然后在blog根目录下打开终端输入命令 cnpm install https://github.com/CodeFalling/hexo-asset-image --save 下载插件

下载好之后，在blog根目录中打开 _config.yml，找到post_asset_folder，把他设置为设为true，在md文件上输入 ![](这里输入图片的相对路径) 此时的图片就可以在网页上打开了，如果没有显示刷新一下试试

例如：我的image文件夹里面的图片，有一张ceshi.png，我要上传名字为 ceshi.png 的图片

那么我在md文件里面就输入 ![](image/ceshi.png) 这样自己的博客就可以看到图片了，刚开始打开博客可能不会显示，刷新一下就可以了

一般部署到GitHub上是没有问题了，但是Gitee可能还会存在一些问题，这时候就要看你的仓库的名字和你路径的名字是否一样，名字强烈推荐自己的博客名（否则可能会有一些莫名其妙的问题），比如我的博客名是mao-design，那么仓库名和路径字也必须是mao-design

当初我就是因为博客名不一样导致图片不能正常显示的，这也是问题之一

不出意外的话，自己博客上就可看到图片了

注意，存放图片文件夹的名称和md文件的名称要一样，也就是只要名字一样随便什么名字都可以

技巧

每次把自己的博客部署到服务器上都要输入那三行命令，很麻烦，所以可以在blog根目录下创建一个bat文件，在bat文件里面输入 hexo clean && hexo g && hexo d 这行代码，以后要部署的时候双击bat文件就可以自动部署了；以此类推，预览也可以这样。
