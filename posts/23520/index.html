部署前准备

TOKEN_GITHUB ：GitHub的 token （私人令牌的密钥只会显示一次，如果忘了，那只能重新创建了）

打开GitHub Token的方法：点击自己的Github头像 --> Settings --> Developer settings --> Personal access tokens --> Tokens(classic) --> Generate new token --> Generate new token(classic)

Note 那里是自己Token的名字，名字可以随便取

Expiration 是自己令牌的有效期，选择 No expiration 永不过期

下面的 Select scopes 那里选择 repo以及其子项，然后在选择 workflow

一切完成之后点击最下面的 Generate token 即可生成自己的 GitHub Token，记住GitHub Token只会显示一次，请务必要记住，否则自能重新生成

TOKEN_GITEE ：Gitee的 token （私人令牌的密钥只会显示一次，如果忘了，那只能重新创建了）

设置Gitee Token的方法：点击自己的Gitee头像，点击设置 --> 点击私人令牌 --> 点击生成新令牌

私人令牌描述就是自己令牌的名字，名字随便取，下面的权限选择 user_info 和 projects

最后点击提交即可生成自己的 Gitee Token，记住Gitee Token只会显示一次，请务必要记住，否则自能重新生成

GIT_NAME ：自己Git名称 ，在设置SSH密钥时使用的Git名称

GIT_EMAIL ：自己的Git邮箱，在设置SSH密钥时使用的Git邮箱

GITEE_PASSWORD ：Gitee 密码

以上的这些信息要部署到自己的GitHub Action仓库中，所以建议将这些信息存放在txt文本中，方便整理添加

关注Gitee公众号：因为GitHub的服务器在国外，所以登录Gitee会提示异地登录验证码；关注Gitee公众号，并且在公众号中绑定自己的Gitee账号，这样每当GitHub登录自己的Gitee时，就不需要异地登陆验证码了，而是直接在公众号中提醒你

GitHub Action权限设置：在GitHub Action的仓库中，打开 Settings --> Actions --> General ，找到 Workflow permissions 选项，选择 Read and write permissions ，以及 Allow GitHub Actions to create and approve pull requests ，然后在点击下面的Save按钮

设置秘密文件：将上面准备的 TOKEN_GITHUB ，TOKEN_GITEE ，GIT_NAME ，GIT_EMAIL ，GITEE_PASSWORD 添加到GitHub Action仓库中的 Repository secrets 中（在Secrets and variables --> Actions 中，找到 Repository secrets 选项，点击 New repository secret 添加），总共五个秘密文件

创建一个用来执行 GitHub Action 的仓库，私有还是公开都可以，推荐私有，这样安全一点

编写GitHub Action配置文件

创建文件夹 .github，然后在 .github 里面创建文件夹 workflows ，在 workflows 中创建文件 xxx.yml ，yml 文件的名字可以随便取，然后在该文件里面编写执行语句

注意yml语句要对齐

基础的配置，选择执行系统，安装博客所需的环境

安装博客所需的插件

如果你不知道gulp插件的用法，建议去看看这篇博客使用gulp压缩博客静态资源 | Akilarの糖果屋

如果你觉得麻烦，可以不下载gulp插件，删掉关于 gulp 的语句即可

如果你的根目录中没有gulp的执行文件（也就是 gulpfile.js ），那么GitHub Action运行gulp时则会报错，这点须知

部署到Git仓库中

脚本最终的效果

部署GitHub Action

将自己的 .github 文件夹和自己的项目文件放在一起，然后一起上传到Github即可，GitHub检测到 .github 会自动执行里面的脚本

点击 Pull requests 旁边的 Actions 即可查看 GitHub Action 的运行情况

GitHub Action运行完成之后显示一个绿色的 ✔ ，则表示运行成功

如果没有运行成功的话，则点击进去查看报错情况，在自己百度，我上面推荐的那些博客说的也比较详细的，推荐去看看
